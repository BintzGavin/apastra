# Community Prompt Pack Acceptance Policy

## Purpose
This policy establishes the governance pathway for accepting, reviewing, and publishing community-contributed prompt packs under a custodian organization.

## Acceptance Criteria
To be accepted, a community prompt pack must meet the following criteria:

1. **Automated Security Scans**: The pack must pass all automated security scans, including secret scanning and vulnerability checks.
2. **Schema Validation**: All files in the pack (prompts, datasets, evaluators, suites) must validate against the official CONTRACTS schemas.
3. **Baseline Dataset & Passing Evals**: The pack must include a baseline dataset and demonstrate passing evaluation scores against defined metrics.
4. **Human Custodian Review**: A designated custodian must review and approve the pack for quality, relevance, and safety.

## Publishing
Upon meeting all criteria and receiving approval, the prompt pack will be appended as an immutable record to the registry.
