# Approachable Terminology Policy

## Overview
This policy governs the use of terminology across user-facing documentation and technical specifications to ensure an approachable experience for new users, in alignment with Refinement 5 of the system vision.

## Rules

1. **User-Facing Contexts**
   - In any user-facing documentation (e.g., guides, tutorials, conceptual overviews), the system must be referred to as "your agent".
   - The term "harness" MUST NOT be used in user-facing contexts.

2. **Technical Contexts**
   - The term "harness" is strictly reserved for technical specifications and deep technical documentation that describes the evaluation or execution environment.
   - "Harness" should only be used when addressing developers actively working on or extending the engine's internal execution model.

3. **Enforcement**
   - All pull requests modifying documentation MUST be reviewed for adherence to this terminology policy.
   - Technical writers and reviewers are responsible for correcting unauthorized uses of "harness" before merging.
