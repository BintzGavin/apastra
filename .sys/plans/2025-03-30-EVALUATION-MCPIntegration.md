#### 1. Context & Goal
- **Objective**: Implement Model Context Protocol (MCP) integration for evaluations.
- **Trigger**: The docs/vision.md outlines "MCP integration" as a proposed refinement to support tool-calling prompt evaluation and allow agents to invoke evals as tools.
- **Impact**: Unlocks native tool-calling evaluation capabilities and enables MCP-compliant IDE agents to discover and execute evaluation suites autonomously.

#### 3. Implementation Spec
- **Harness Architecture**: Create an MCP server (`mcp-adapter/server.py`) that acts as an adapter. It will expose a `run_suite` tool that accepts minimal parameters (e.g. suite ID and revision). When invoked, it builds a full local `run_request.json` meeting the schema requirements, calls the existing `promptops/runs/runner-shim.sh`, and returns a text summary of the execution by parsing the generated `run_manifest.json` and any normalized metrics extracted.
- **Run Request Format**: Unchanged. The MCP adapter translates tool arguments into the standard `run_request.json` format, supplying necessary placeholder digests if required by the schema.
- **Run Artifact Format**: Unchanged. The MCP adapter reads the `run_manifest.json` generated by the underlying harness to formulate its string response to the agent.
- **Pseudo-Code**:
  - Import required tools (e.g., standard library `json`, `subprocess`, `sys`, and the `mcp` package).
  - Define an MCP server instance.
  - Register a tool endpoint `run_evaluation(suite_id, prompt_ref)` that:
    - Creates a temporary directory.
    - Generates a valid `run_request.json` file inside it.
    - Uses `subprocess.run` to execute `bash promptops/runs/runner-shim.sh <adapter_yaml> <run_request> <output_dir>`.
    - Reads `<output_dir>/run_manifest.json` and parses relevant metrics outputs.
    - Formats and returns a summary string of the metrics and status.
  - Start the server listening on stdio.
- **Baseline and Regression Flow**: The tool can optionally accept a baseline reference and return regression status by orchestrating the existing `promptops/runs/generate_regression_report.sh` script.
- **Dependencies**: Requires the Python `mcp` package (must be installed via pip in the environment). Depends on the schemas for `harness-adapter.schema.json`, `run-request.schema.json`, `run-manifest.schema.json`, and `scorecard.schema.json`.
