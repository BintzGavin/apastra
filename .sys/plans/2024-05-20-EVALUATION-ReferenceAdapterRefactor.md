#### 1. Context & Goal
- **Objective**: Refactor the reference harness adapter to output split artifact files and correctly invoke the updated Scorecard Normalizer.
- **Trigger**: The recent architecture shift to append-friendly split artifacts and the refactored `normalize.py` broke the reference adapter's compatibility, causing it to output a monolithic artifact.
- **Impact**: Unlocks successful execution of the reference harness via the RUNTIME runner shim, ensuring it complies with the latest CONTRACTS schemas and the split-artifact vision.

#### 2. File Inventory
- **Create**: None
- **Modify**: `promptops/harnesses/reference-adapter/run.py` (Update to write split files directly and pass correct args to `normalize.py`)
- **Read-Only**: `promptops/runs/normalize.py`, `promptops/runtime/runner.py`, `promptops/schemas/run-manifest.schema.json`

#### 3. Implementation Spec
- **Harness Architecture**: The reference adapter must natively implement the updated contract (input: run request; output: split run artifact files: `run_manifest.json`, `cases.jsonl`, `failures.json`, `artifact_refs.json`, and `scorecard.json`).
- **Run Request Format**: Key fields required (prompt digest, dataset digest, model config, harness version, sampling config).
- **Run Artifact Format**: Split into separate files. `run_manifest.json` (metadata, status), `cases.jsonl` (per-case output), `failures.json` (if any), `artifact_refs.json`. `scorecard.json` will be generated by `normalize.py`.
- **Pseudo-Code**:
  1. Read `run_request.json` from `sys.argv[1]`.
  2. Perform evaluation logic (mock resolver, etc.).
  3. Write `run_manifest.json` to `output_dir`.
  4. Write `cases.jsonl` to `output_dir` by iterating over cases.
  5. Write `failures.json` to `output_dir`.
  6. Write `artifact_refs.json` to `output_dir` with a dummy digest.
  7. Call `normalize.py` with arguments `output_dir/cases.jsonl` and `output_dir/scorecard.json`.
- **Baseline and Regression Flow**: Baselines and regression reports depend on the distinct `scorecard.json` generated by this refactored flow.
- **Dependencies**: CONTRACTS schemas required (`run-manifest.schema.json`, `run-case.schema.json`, `scorecard.schema.json`); RUNTIME resolver availability; Scorecard Normalizer (`normalize.py`).

#### 4. Test Plan
- **Verification**: Run `python promptops/runtime/runner.py <mock_run_request.json> promptops/harnesses/reference-adapter/adapter.yaml <output_dir>`
- **Success Criteria**: The command succeeds with exit code 0, and the `output_dir` contains strictly compliant `run_manifest.json`, `cases.jsonl`, `failures.json`, `artifact_refs.json`, and `scorecard.json` files.
- **Edge Cases**: Evaluator failures or missing prompt digest should be handled gracefully and written to `failures.json`.
