#### 1. Context & Goal
- **Objective**: Implement multi-model comparison generation.
- **Trigger**: docs/vision.md explicitly lists "Multi-model comparison" (Expansion 3) as a feature, which requires running a suite against N models simultaneously to produce a comparison scorecard with per-model breakdowns and tradeoff surfaces.
- **Impact**: Enables teams to evaluate and compare prompts across multiple models, significantly advancing the evaluation capabilities as planned in the vision document.

#### 2. File Inventory
- **Create**: promptops/runs/generate_comparison_scorecard.py
- **Modify**: None
- **Read-Only**: docs/vision.md, promptops/runs/compare.py

#### 3. Implementation Spec
- **Harness Architecture**: Spec a new utility script or command that iterates through a model_matrix within a suite, invoking the runner shim for each model.
- **Run Request Format**: The run_request.json already contains model_matrix. The implementation will resolve it and run the specified adapter iteratively or concurrently for each model.
- **Run Artifact Format**: A combined comparison_scorecard.json will be generated by aggregating outputs from individual scorecard.json artifacts, providing per-model breakdowns.
- **Pseudo-Code**:
  ```bash
  #!/bin/bash
  # Iteratively invoke harness for each model
  for MODEL in \${MODELS[@]}; do
      # Generate run request for specific model
      ./runner-shim.sh ...
  done
  # Aggregate scorecards
  python generate_comparison_scorecard.py ...
  ```
- **Dependencies**: Depends on existing execution scripts (runner-shim.sh) and CONTRACTS schemas.

#### 4. Test Plan
- **Verification**: Run the execution against a test suite with multiple models specified in the model matrix.
- **Success Criteria**: A single comparison_scorecard.json is generated containing evaluation metrics segmented by model.
- **Edge Cases**: Missing scorecards, variable evaluator output across models, handling model failure during parallel execution.
