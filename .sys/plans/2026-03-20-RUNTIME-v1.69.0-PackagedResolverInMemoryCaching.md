#### 1. Context & Goal
- **Objective**: Implement in-memory caching in `PackagedResolver` using the requested digest or reference as the cache key.
- **Trigger**: The current implementation of `PackagedResolver` caches remote fetches to disk, but lacks in-memory caching to prevent redundant file I/O and JSON/YAML parsing when the same packaged artifact is resolved multiple times in a single session.
- **Impact**: Improves performance by reducing redundant file I/O, aligning with the in-memory caching strategies implemented in other resolvers.

#### 2. File Inventory
- **Create**: None
- **Modify**: `promptops/resolver/packaged.py`
- **Read-Only**: `docs/vision.md`, `README.md`, `promptops/schemas/prompt-package.schema.json`, `promptops/schemas/provider-artifact.schema.json`

#### 3. Implementation Spec
- **Resolver Architecture**: Add an `__init__(self)` method to `PackagedResolver` that initializes an in-memory `self.cache = {}` dictionary. The `resolve(self, prompt_id, ref)` function will first check if `(prompt_id, ref)` exists in `self.cache`. If present, it returns the cached result. Otherwise, it executes the existing resolution logic, stores the final resolved `prompt_spec` (for sha256) or recursive result (for provider artifacts) in `self.cache[(prompt_id, ref)]`, and then returns it.
- **Manifest Format**: N/A
- **Pseudo-Code**:
  - `__init__(self)`: initialize `self.cache = {}`.
  - `resolve(self, prompt_id, ref)`:
    - check `if (prompt_id, ref) in self.cache: return self.cache[(prompt_id, ref)]`
    - execute existing fetch and schema validation logic
    - `self.cache[(prompt_id, ref)] = resolved_spec_or_recursive_result`
    - `return self.cache[(prompt_id, ref)]`
- **Harness Contract Interface**: N/A
- **Dependencies**: Depends on the existing CONTRACTS schemas (`prompt-package.schema.json` and `provider-artifact.schema.json`) for validation.

#### 4. Test Plan
- **Verification**: Instantiate `PackagedResolver`, resolve a known local or remote artifact reference twice, and verify that the in-memory cache is populated and accessed on the second call.
- **Success Criteria**: The second call returns exactly the same object from memory, and no new file I/O occurs.
- **Edge Cases**: Ensure invalid refs still throw appropriate errors and are not cached.
